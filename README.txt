Якщо дозволена міні-автоматизація — використовуй будь-який include (наприклад, Nunjucks/Handlebars локально) і потім збирай у єдиний HTML, але у клієнт віддавай один файл.

Шрифти:
- Найбезпечніше: system-ui, -apple-system, Segoe UI, Roboto, Arial, sans-serif.
- Якщо потрібен бренд-шрифт — підключай @font-face у <style> з власного CDN (woff2/woff). Працює у Apple Mail/iOS Mail, у Gmail/Outlook — буде fallback.

Темна тема:
- Уже ввімкнено через @media (prefers-color-scheme: dark) + [data-ogsc] (Outlook.com).
- Для логотипів є components/dark-mode-logo-swap.html — показує різні зображення для світлої/темної теми.

Зображення:
- Ретина: роби вихідні у 2x, але став width/height за макетом і style="height:auto".
- CDN/HTTPS. ALT обов’язково. Для бекґраунду у герої — дивись components/hero-bg-vml.html.

Кнопки:
- Тільки bulletproof: VML для Outlook + звичайний <a> для інших. Дивись components/button.html.

Респонс:
- В основі — таблиці. Стек колонок на мобільному через .sm-block + медіа-запит (max-width:600px).

Обмеження та підводні камені:
- Gmail може "обрізати" контент > ~102KB. Стискай HTML, не зловживай коментарями/inline-стилями.
- Outlook (Windows) рендерить Word’ом: уникай складних CSS. Лейаути — тільки <table>.
- Background-image у Outlook Desktop працює лише через VML.
- Не викорінюй <style> зовсім — але ключові стилі тримай інлайном.

Доставність і аналітика:
- Додавай UTM до посилань (через CRM або вручну).
- Трек-піксель — тільки якщо дозволено політикою.
- Обов’язкові посилання: відписка, адреса компанії, політика.
